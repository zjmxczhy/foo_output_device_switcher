# foobar2000 无障碍增强组件使用方法

`foobar2000 无障碍增强` 是一个 foobar2000 组件，用于在播放菜单中查看和切换 foobar2000 使用的播放设备，也可以切换 foobar2000 的播放模式、播放列表，并通过屏幕阅读器/Tolk 或 SAPI5.1 播报当前设备、切换后的设备名称、切换后的播放模式或播放列表名称。组件还提供启动后隐藏 foobar2000 主窗口的设置，用于减少托盘模式下读屏读到幽灵窗口的问题。

## 适用环境

- Windows
- foobar2000 2.x，支持 x64 和 x86 版本
- foobar2000 1.5 / 1.6，使用 x86 兼容包

## 安装

请按 foobar2000 的架构选择安装包：

```text
foo_output_device_switcher-0.2.2-x64.fb2k-component
foo_output_device_switcher-0.2.2-x86.fb2k-component
```

foobar2000 2.x 的 x64 版本请安装 `x64` 包；foobar2000 2.x 的 x86 版本请安装 `x86` 包。

foobar2000 1.5 / 1.6 是 32 位程序，请安装 `x86` 包。后续 `x86` 包默认用于兼容 foobar2000 1.5 / 1.6。

在 foobar2000 中打开：

```text
文件 -> 参数选项 -> 组件
```

选择组件包安装，安装完成后重启 foobar2000。

## 菜单位置

安装后，菜单位置为：

```text
播放 -> 播放设备
播放 -> 播放模式
播放 -> 播放列表
```

`播放设备` 子菜单中包含：

```text
查看当前播放设备
切换到上一个播放设备
切换到下一个播放设备
```

`播放模式` 子菜单中包含：

```text
查看当前播放模式
切换到上一个播放模式
切换到下一个播放模式
```

`播放列表` 子菜单中包含：

```text
查看当前播放列表
切换到上一个播放列表并播放
切换到下一个播放列表并播放
```

## 使用方法

选择 `查看当前播放设备`，组件会播报 foobar2000 当前使用的播放设备。

播报格式为：

```text
当前 : 设备名称
```

选择 `切换到下一个播放设备`，foobar2000 会切换到设备列表中的下一个播放设备。

选择 `切换到上一个播放设备`，foobar2000 会切换到设备列表中的上一个播放设备。

设备列表会循环切换。例如当前已经是最后一个设备时，继续切换下一个会回到第一个设备。

选择 `查看当前播放模式`，组件会播报 foobar2000 当前使用的播放模式。

选择 `切换到下一个播放模式`，foobar2000 会切换到当前播放模式列表中的下一个模式。

选择 `切换到上一个播放模式`，foobar2000 会切换到当前播放模式列表中的上一个模式。

播放模式列表来自 foobar2000 自身，例如默认、重复、随机、单曲循环等；如果 foobar2000 或其他组件提供了不同的播放模式，菜单会按 foobar2000 当前返回的列表循环切换。

选择 `查看当前播放列表`，组件会播报 foobar2000 当前活动播放列表。

选择 `切换到下一个播放列表并播放`，foobar2000 会切换到播放列表列表中的下一个播放列表，并播放该播放列表中的焦点歌曲；如果没有焦点歌曲，则播放第一首歌曲。

选择 `切换到上一个播放列表并播放`，foobar2000 会切换到播放列表列表中的上一个播放列表，并播放该播放列表中的焦点歌曲；如果没有焦点歌曲，则播放第一首歌曲。

播放列表会循环切换。例如当前已经是最后一个播放列表时，继续切换下一个会回到第一个播放列表。

## 播报行为

切换成功后，组件会直接播报切换后的设备名称。

例如切换到耳机设备时，播报内容类似：

```text
扬声器 (Realtek(R) Audio)
```

查看当前播放设备时，播报内容类似：

```text
当前 : 扬声器 (MUV HW 1/2)
```

切换播放模式时，播报内容类似：

```text
播放模式 : 随机
```

查看当前播放模式时，播报内容类似：

```text
当前 : 随机
```

切换播放列表并播放时，播报内容类似：

```text
播放列表 : 默认
```

查看当前播放列表时，播报内容类似：

```text
当前 : 默认
```

## 参数设置

打开：

```text
文件 -> 参数选项 -> 工具 -> foobar2000 无障碍增强
```

`foobar2000 无障碍增强` 是一个可展开分组，下面包含：

```text
常规设置
TTS 语音
关于
```

### 常规设置

打开：

```text
文件 -> 参数选项 -> 工具 -> foobar2000 无障碍增强 -> 常规设置
```

当前可设置：

```text
启动后隐藏 foobar2000 主窗口
启动后隐藏主窗口延迟毫秒
```

#### 启动后隐藏 foobar2000 主窗口

用于解决 foobar2000 首次启动后，托盘模式下读屏仍能读到主窗口标题的幽灵窗口问题。

- 选中：组件启动后会延迟调用 foobar2000 的隐藏主窗口命令，并在短时间内复查窗口状态。
- 未选中：不处理启动后的主窗口隐藏。

默认未选中。

如果你以前在 `朗读 LRC 歌词` 组件里勾选过这个功能，`foobar2000 无障碍增强` 会继续读取原来的设置值。

#### 启动后隐藏主窗口延迟毫秒

用于控制首次启动后多久隐藏主窗口。

默认值：

```text
800
```

可填写范围：

```text
0 到 10000
```

如果首次启动后仍能短暂读到幽灵窗口，可以尝试改成：

```text
100
300
1500
2000
```

### TTS 语音

打开：

```text
文件 -> 参数选项 -> 工具 -> foobar2000 无障碍增强 -> TTS 语音
```

当前可设置：

```text
使用屏幕阅读器
选择语音
语速
试听文本
试听
```

#### 使用屏幕阅读器

- 选中：组件通过屏幕阅读器/Tolk 播报菜单操作结果。
- 未选中：组件改用 Windows SAPI5.1 语音播报。

默认选中。

#### 选择语音

取消勾选 `使用屏幕阅读器` 后，可以在 `选择语音` 下拉框里选择系统已安装的 SAPI5.1 语音。

如果不选择具体语音，组件会使用系统默认 SAPI5.1 语音。

#### 语速

`语速` 用于调整 SAPI5.1 播报速度，范围是：

```text
-10 到 10
```

默认值是：

```text
0
```

#### 试听

取消勾选 `使用屏幕阅读器` 后，可以修改 `试听文本`，再按 `试听` 按钮测试当前 SAPI5.1 语音和语速。

`试听文本` 是多行编辑框，可以用 Tab 切入编辑。按 Tab 会继续切到下一个控件，按 Shift+Tab 会回到上一个控件，不会把焦点锁在编辑框里。

### 关于

打开：

```text
文件 -> 参数选项 -> 工具 -> foobar2000 无障碍增强 -> 关于
```

`关于` 页面包含：

```text
主要用途
查看 GitHub 仓库
前往 GitHub 发布页
备用 gitee 发布页
```

`主要用途` 是只读编辑框，可以用 Tab 切入让屏幕阅读器读取内容。按 Tab 会继续切到下一个按钮，按 Shift+Tab 会回到上一个控件，不会把焦点锁在编辑框里。

三个按钮会调用系统默认浏览器打开对应页面：

```text
GitHub 仓库：https://github.com/zjmxczhy/foo_output_device_switcher
GitHub 发布页：https://github.com/zjmxczhy/foo_output_device_switcher/releases
备用 gitee 发布页：https://gitee.com/zjmxczhy/foo_output_device_switcher/releases
```

## Tolk 文件

组件包中已经包含所需的 Tolk 文件，来源为：

```text
https://github.com/boz700908/tolk
```

安装组件包后不需要手动复制 `Tolk.dll`。

## 注意事项

- 本组件切换的是 foobar2000 的播放输出设备，不会修改 Windows 系统默认声卡。
- 如果 foobar2000 当前正在播放，切换设备可能会触发 foobar2000 重启输出流。
- 如果只有一个可用播放设备，菜单命令会显示为不可用。
- 如果只有一个可用播放模式，播放模式切换命令会显示为不可用。
- 如果只有一个播放列表，播放列表切换并播放命令会显示为不可用。
- 如果目标播放列表为空，组件会播报 `播放列表为空 : 播放列表名称`，不会开始播放。
- 如果无法识别当前播放设备，组件会播报无法确定当前播放设备。
- 启动后隐藏主窗口设置在下次启动 foobar2000 时生效。
- 如果 foobar2000 是通过打开 `.fb2k-component` 安装包启动的，组件会自动跳过本次启动隐藏，避免影响组件安装确认窗口。
- 如果没有听到播报，请确认屏幕阅读器或 ZDSR 相关运行环境正常。

## 重新构建

在仓库根目录运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\build_output_device_switcher.ps1
```

构建完成后会重新生成：

```text
dist\foo_output_device_switcher-0.2.2-x64.fb2k-component
dist\foo_output_device_switcher-0.2.2-x86.fb2k-component
```
